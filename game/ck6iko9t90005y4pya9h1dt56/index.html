transmission报错"permission denied"的一种解决方法

发表于 2020.01.30 分类于 游戏 Valine：

记录一下transmission报错”permission denied”的一种解决方法

系统：cent os7
安装transmission用的命令：yum install transmission-cli transmission-common transmission-daemon
默认下载路径：/var/lib/transmission/Download
修改后下载路径：/root/Files
问题原因：没权限

分析过程：

  1. 查看transmission所属群组(一般是transmission)：找到transmission-daemon路径用ls-ll
  2. 查看目标下载路径所属群组(这里是root)：找到/root/Files用ls-ll看Files对应条目
  3. 发现群组不同，对应权限中的”其它组”权限，即把Files的”其它组”权限改为rwx(用chmod -R,第三位是7)
  4. 在Files之前所有父目录的”其它组”权限中加入”可执行”权限(rwx中的x)，让transmission能从根目录正常地进入目标下载路径(“可执行”权限使用户能进入某目录或执行某文件)

实际操作：

验证：
进入transmission的添加任务窗口，发现”下载路径”输入框的下面显示出文件夹剩余容量，添加任务后顺利下载，说明问题已解决。
