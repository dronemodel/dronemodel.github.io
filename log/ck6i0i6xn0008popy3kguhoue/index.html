建站过程

发表于 2019.09.09 更新于 2019.09.13 分类于 日志 Valine：

简单写一下我建立blog的经历。

假期里到各大博物馆看了看，回来就想写点东西。一开始先搜了一下wordpress怎么建站，因为我那时只知道这个，然而在搜索过程中我看到有很多人推荐hexo，轻便快捷。就改选了hexo。建站过程都是跟着教程做的，教程也是网上随便找的，困难在于路上遇到的各种麻烦，我简单写一下。

1.安装了nodejs后查看版本，可以查看，但是转到blog路径使用npm时却“找不到命令”。期间尝试了重设环境变量、换cmd操作、手动cd到路径、用管理员模式打开git bash等方法，最后把我整晕了之后偶然成功，我也没有精力记录解决方案是什么。

2.安装hexo时报出大量warn，不能继续安装（即装完后找不到hexo指令，没有版本号）。搜索后把warn内容提示的几个依赖包更新了，并使用cmd完成的安装。用git bash应该也能装，我没试。不过经历了两次安装问题后我对git bash好感大幅下降，再加上界面字体小，安装过程没有高亮提示，不显示安装进度，我已经不想用git bash了。

3.ssh-key上传到github之后验证是否添加成功，没成功。其实问题不大，按照ssh -T的提示继续操作就好，不要移动.ssh文件夹。好像是.ssh文件夹里缺少一个文件造成的。

4.域名解析的时候注意，我ping错域名了，应该看yourname.github.io的ip。我查的是我个人域名的ip。这导致github设置页面里提示站点没有上传到域名。

5.之后就是各种配置个性化，加入各种插件。因为同类型插件往往有多种，我的策略是一种不会用就换另一种，勉强是配完了网站。

建站之后

1.markdown里链接语法是 [描述]:链接 ，可是我加了之后在generate阶段并没有生成链接的html语句，站点页面里自然不会有链接，整个语句连同链接描述都消失了。简单说，插入链接失败。多次尝试后发现，如果同一行里方括号 [ ] 之前有其他字符就可以显示语句，如果 [ ] 与 : 之间有空格也可以，总之要把插入链接的语句独立起来。但是效果只是链接文本本身可以点击，链接描述还是普通文本格式，还不如不用链接语法。不知道是什么原理。

2.很气。重新安装了hexo。这次更新了core-js，更新了hexo-renderer-marked@2.0.0，还是没用。仔细比对了自带的hello world文件，原来是语法用错了。插入链接是 [描述] (链接) 中间没空格。之前用的是参考形式，文章中如果没有引用就不显示。😢

3.因为加了文件管理站，为了统一域名等级，把原来在二级域名下的页面放到了www下，重设了域名解析，结果被墙了，不懂。改回来了。又把nameserver改到cloudflare那家，看介绍它提供免费ssl，搞了半天，没成。改回来了。

4.把解析记录全删了又配一遍，感觉对一些操作开始理解了。终于搞定，移动端也可以进入网站了。顺便升级了下载站（支持https）。发现个bug，github的custom page修改后短时间内有可能自动改回去。
